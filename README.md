# 問い合わせ・予約フォーム 自動さばきシステム（Google Apps Script）

Googleフォームに届いた問い合わせを、**人が触らずに**「台帳へ整形 → 申込者へ自動返信 → 社内へ通知 → 毎朝レポート」まで自動でさばく仕組みです。
Googleアカウントだけで動き、サーバー費用は **0円**。IT担当がいない中小企業・個人店でもそのまま導入できます。

---

## できること

| # | 自動でやること | 効果 |
|---|----------------|------|
| 1 | フォーム回答を **受付番号付き**で管理台帳に整形・追記 | 転記ミス・記入漏れの確認が不要に |
| 2 | 申込者へ **自動返信メール**（受付番号入り） | 「届いていますか？」の電話が減る |
| 3 | 社内へ **Slack / メールで即時通知** | 対応の取りこぼし・遅れを防止 |
| 4 | 毎朝 **日次レポート**（前日件数・分類内訳・未対応残）を自動配信 | 状況把握の手作業がゼロに |
| 5 | 処理が失敗したら **管理者へ自動でエラー通知** | 「気づいたら止まっていた」を防ぐ |

> **Before / After** の具体的な効果は [`docs/before-after.md`](docs/before-after.md) を参照。

## 構成

```
gas-form-autopilot/
├── appsscript.json        … プロジェクト設定（タイムゾーン等）
├── src/
│   ├── Config.gs          … ★設定はここだけ編集すればOK
│   ├── Code.gs            … メイン処理（フォーム送信トリガー・受付番号採番）
│   ├── Notify.gs          … 自動返信メール / Slack / 社内メール通知
│   ├── Report.gs          … サブ機能：日次レポート自動配信
│   └── Common.gs          … 共通処理（エラー通知）
├── docs/
│   └── before-after.md    … 導入効果（営業・提案用）
└── sample/
    └── sheet-structure.md … デモ用のフォーム・台帳の構成
```

**設計のポイント**
- **設定とロジックを分離**：`Config.gs` だけ変えれば文面・通知先・列名を変更でき、本体コードは触らせない。
- **機密情報をコードに書かない**：Slack の Webhook URL は「スクリプトプロパティ」に保管し、公開リポジトリに漏れない設計。
- **失敗を検知できる**：エラー時に管理者へ自動通知。実運用で一番大事な部分。

## 初期設定（5ステップ・約10分）

### 1. Googleフォームとスプレッドシートを用意
1. 受付に使う **Googleフォーム**を作成（例：お名前 / メールアドレス / ご相談内容）。
2. フォームの「回答」タブ →「スプレッドシートにリンク」で回答用シートを作成。

### 2. スクリプトを貼り付け
1. リンクしたスプレッドシートで `拡張機能 → Apps Script` を開く。
2. `src/` の中身（`Config.gs` `Code.gs` `Notify.gs` `Report.gs` `Common.gs`）を、それぞれ同名ファイルとして貼り付ける。
   - ファイル追加は Apps Script エディタ左の「＋ → スクリプト」。

### 3. 設定を自分用に変更（`Config.gs`）
- `COLUMNS` … フォームの質問タイトルに合わせる（例：`NAME: 'お名前'`）。
- `AUTO_REPLY` … 自動返信メールの件名・本文・差出人名。
- `MANAGEMENT_SHEET_NAME` … 台帳シート名（そのままでOK）。

### 4. Slack通知を使う場合（任意）
1. Slackで [Incoming Webhook](https://api.slack.com/messaging/webhooks) を作成し、URLをコピー。
2. Apps Scriptエディタ → `プロジェクトの設定（歯車）` → `スクリプト プロパティ` →
   - プロパティ名：`SLACK_WEBHOOK_URL`
   - 値：コピーしたWebhook URL
3. Slackを使わない場合は `Config.gs` の `SLACK.ENABLED` を `false`、`STAFF_EMAIL.ENABLED` を `true` にしてメール通知に切替。

### 5. トリガーを登録
1. Apps Scriptエディタで関数 `setupTriggers` を選び ▶ 実行。
2. 初回は権限の承認画面が出るので許可（自分のGoogleアカウント内で動くだけです）。
3. これで「フォーム送信時」と「毎朝の日次レポート」が自動で動き始めます。

## 動作テスト
- フォームからテスト送信 → 台帳に行が増え、自動返信メールが届き、Slack/メール通知が来ればOK。
- エディタから手動で確認したい場合は、回答が1件ある状態で `onFormSubmit` を実行（最新回答を読んで処理します）。

## よくある質問

**Q. 費用はかかりますか？**
A. かかりません。Googleアカウントの無料枠で動きます（メール送信は1日あたりの上限あり：無料アカウントで100通/日、Workspaceで1,500通/日）。

**Q. プログラミングの知識は必要ですか？**
A. 導入は手順どおりで完了します。文面や通知先の変更は `Config.gs` の該当箇所を書き換えるだけです。

**Q. セキュリティは大丈夫ですか？**
A. データは依頼者ご自身のGoogleアカウント内に保存され、外部サーバーには送りません。Slack URL などの機密情報もコードには書かない設計です。
