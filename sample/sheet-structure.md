# デモ用のフォーム・シート構成

ポートフォリオのデモや動作確認を、依頼者の実データなしで再現するための構成例です。

## 1. Googleフォーム（受付フォーム）の質問例

| 質問タイトル | 種類 | 必須 |
|--------------|------|------|
| お名前 | 記述（短文） | ○ |
| メールアドレス | 記述（短文） | ○ |
| ご相談内容 | プルダウン（料金/予約/その他 など） | ○ |
| 詳細・ご要望 | 記述（長文） | － |

> `Config.gs` の `COLUMNS` を、ここの質問タイトルと一致させてください。

## 2. 回答シート（フォームが自動生成）

`フォームの回答 1` という名前で自動作成されます。
1列目に「タイムスタンプ」、以降に各質問の回答が入ります。**ここは触りません。**

## 3. 管理台帳シート（スクリプトが自動生成）

`問い合わせ管理` シートが自動で作られ、以下の形で1件1行が追記されます。

| 受付番号 | 受付日時 | 氏名 | メール | 分類 | 対応状況 | 内容（元データ） |
|----------|----------|------|--------|------|----------|------------------|
| REQ-20260614-001 | 2026/06/14 10:32 | 山田 太郎 | taro@example.com | 予約 | 未対応 | {"お名前":"山田 太郎", ...} |

### 運用のおすすめ
- **対応状況**の列はプルダウン（`未対応` / `対応中` / `対応済`）にしておくと、
  日次レポートの「未対応の残件」が正しく数えられます。
  - 設定方法：列を選択 → `データ → データの入力規則 → プルダウン`

## 4. デモのやり方
1. 上記フォームを作ってスプレッドシートにリンク。
2. README の手順でスクリプトを設置し `setupTriggers` を実行。
3. フォームから2〜3件テスト送信。
4. 台帳への追記・自動返信メール・Slack通知・翌朝のレポートを確認。
